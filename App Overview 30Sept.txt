Algorithm Overview
Located in matchingEngine.ts, this is a sophisticated scoring system with hard filters and weighted features.
Hard Filters (Eliminatory)
Language Overlap: â‰¥1 shared language required
Timezone Distance: â‰¤3 hours difference maximum
Mentor Capacity: >0 capacity_remaining required
Scoring Formula (0-100 points)
Total Score = MAX(0, MIN(100,
  40 Ã— topics_overlap +          // Jaccard similarity
  15 Ã— industry_overlap +         // Binary match (1 if same)
  10 Ã— role_seniority_fit +       // Seniority gap scoring
  20 Ã— semantic_similarity +      // Text analysis of goals
  5 Ã— tz_overlap_bonus +          // â‰¤2 hours = bonus
  5 Ã— language_bonus -            // Primary language match
  10 Ã— capacity_penalty           // =1 if mentor at capacity
))
Feature Calculations
Topics Overlap (40 points max)
Uses Jaccard similarity: intersection / union
Compares mentee's topics_to_learn vs mentor's topics_to_mentor
Industry Overlap (15 points max)
Currently returns 1.0 (assumes same company = same industry)
Future: Extract from role or dedicated field
Role Seniority Fit (10 points max)
Uses SENIORITY_SCORES mapping (IC1-IC6, M1-M6)
Rewards appropriate seniority gap
Penalty if mentor less senior than mentee
Semantic Similarity (20 points max)
Keyword-based text analysis
Compares mentee goals vs mentor bio + topics
Simple implementation (filters words >3 chars, finds overlap)
Timezone Bonus (5 points max)
1.0 if â‰¤2 hours difference, else 0
Language Bonus (5 points max)
1.0 if mentor speaks mentee's primary language
Capacity Penalty (-10 points max)
-10 points if mentor has only 1 slot remaining
Tie-Breaking Logic
Total score (descending)
Topics overlap (descending)
Semantic similarity (descending)
Capacity remaining (descending)
Mentor name Aâ†’Z
Matching Modes
Batch Mode
Assigns each mentee to single best mentor
Greedy algorithm: decreases mentor capacity after each assignment
Stops when mentors reach capacity
Top 3 Mode
Returns 3 best mentor recommendations per mentee
Doesn't reduce capacity (exploration mode)
Admins review and manually assign
